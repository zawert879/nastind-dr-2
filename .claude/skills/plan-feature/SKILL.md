# /plan-feature — Спланировать фичу

## Описание
Декомпозировать фичу на задачи до написания кода.

## Вход
Описание фичи от пользователя.

## Шаги

1. Прочитай `.claude/rules/prd-summary.md` — соответствует ли фича продукту
2. Прочитай `.claude/memory/dev-plan.md` — где фича в плане
3. Прочитай `.claude/memory/architecture-decisions.md` — нет ли ограничений
4. Переключись в режим Orchestrator (`.claude/agents/orchestrator.md`)
5. Декомпозируй фичу:
   - Какие компоненты нужны (новые / изменённые)
   - Какие stores нужны
   - Какие API-вызовы нужны
   - Какие роуты нужны
6. Определи порядок реализации
7. Обнови `dev-plan.md` — добавь задачи

## Ограничения

- Не писать код на этом этапе
- Помнить: только фронтенд, без серверных файлов
- API-эндпоинты считать существующими (бэкенд отдельно)

## Формат вывода

```
## Фича: [название]

### Описание
<!-- что и зачем -->

### Компоненты
- [ ] ComponentName — описание

### Stores
- [ ] storeName — описание

### API-вызовы
- [ ] GET /endpoint — описание

### Роуты
- [ ] /path — описание

### Порядок реализации
1. ...
2. ...

### Обновлён dev-plan.md ✓
```
