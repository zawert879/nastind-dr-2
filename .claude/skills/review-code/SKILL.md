# /review-code — Ревью кода

## Описание
Провести код-ревью перед коммитом.

## Шаги

1. Посмотри изменённые файлы (`git diff` / `git status`)
2. Переключись в режим Reviewer (`.claude/agents/reviewer.md`)
3. Проверь каждый файл по чеклисту из `reviewer.md`
4. Выдай результат ревью

## На что смотреть

### Критические
- Серверные файлы (`+page.server.ts`, `+server.ts`) — их быть не должно
- XSS (использование `@html`)
- Прямые `fetch` вместо `src/lib/api/`
- `any` в TypeScript
- Утечки чувствительных данных

### Качество
- Соответствие `svelte-style-guide.md`
- Переиспользуемость компонентов
- Корректная типизация
- Отсутствие закомментированного кода

## Формат вывода

Используй формат из `.claude/agents/reviewer.md`.
