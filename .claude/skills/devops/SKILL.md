# /devops — DevOps задачи

## Описание
Запустить DevOps-агента для задач сборки, деплоя, CI/CD, Docker и инфраструктуры.

## Вход
Описание задачи: проблема сборки, настройка деплоя, оптимизация CI, вопрос по Docker и т.д.

## Шаги

1. Прочитай `.claude/memory/dev-plan.md` — текущий контекст проекта
2. Прочитай `.claude/rules/project-reference.md` — стек (SvelteKit + adapter-static)
3. Прочитай `.claude/memory/architecture-decisions.md` — принятые решения
4. Изучи `package.json` — скрипты и зависимости
5. Переключись в режим DevOps (`.claude/agents/devops.md`)
6. Выполни задачу:
   - Диагностика: определи корневую причину
   - Решение: минимальные, воспроизводимые изменения
   - Проверка: команды для верификации результата
7. Обнови `dev-plan.md` если задача входит в план

## Типичные задачи

- `npm run build` падает — диагностика и фикс
- Настройка GitHub Actions для auto-deploy
- Оптимизация размера бандла
- Dockerfile для локальной разработки
- Настройка переменных окружения
- Анализ `npm audit` и обновление зависимостей
- Настройка деплоя на Netlify / Vercel / GitHub Pages

## Ограничения

- Только фронтенд-инфраструктура (нет бэкенда)
- Деплой только статики через `adapter-static`
- Деструктивные операции (force push, прод-деплой) — с подтверждением пользователя
- Секреты не хранить в файлах репо
