# /update-docs — Синхронизировать документацию

## Описание
Проверить и обновить все файлы документации после изменений в коде.

## Шаги

1. Прочитай `git diff` или посмотри последние изменения
2. Проверь и обнови (если нужно):
   - `.claude/memory/dev-plan.md` — отметить выполненные задачи
   - `.claude/memory/architecture-decisions.md` — новые решения
   - `.claude/memory/tasks-completed.md` — перенести завершённые задачи
   - `.claude/memory/known-bugs.md` — новые/исправленные баги
   - `.claude/rules/project-reference.md` — если изменился стек/структура
   - `.claude/rules/prd-summary.md` — если изменились требования
3. Выдай отчёт что обновлено

## Формат вывода

```
## Документация обновлена

### Изменённые файлы
- dev-plan.md — [что изменилось]
- ...

### Не требовали обновления
- ...

### Требуют внимания
- ...
```
