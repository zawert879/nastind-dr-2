# /commit — Закоммитить изменения

## Описание
Сформировать осмысленный git-коммит по текущим изменениям и обновить память проекта.

## Шаги

1. Запусти `git status` — посмотри какие файлы изменены
2. Запусти `git diff` — прочитай изменения (не более 200 строк)
3. Запусти `git log --oneline -5` — проверь стиль коммитов в репозитории
4. Сформируй сообщение коммита:
   - Первая строка: `тип: краткое описание` (до 72 символов)
   - Типы: `feat`, `fix`, `refactor`, `test`, `chore`, `docs`
   - Язык: русский или английский — следуй стилю существующих коммитов
5. Добавь файлы в staging: только релевантные (не `.env`, не бинарники)
6. Создай коммит
7. Выведи итог: хэш коммита, список файлов, сообщение

## Правила

- Никогда не коммить: `.env`, секреты, `node_modules`, бинарные файлы > 1MB
- Не использовать `git add .` без проверки `git status`
- Не пушить автоматически — только локальный коммит
- Если изменений нет — сообщи об этом
- Если изменения касаются нескольких несвязанных фич — предупреди и спроси, делать один коммит или несколько

## Формат вывода

```
## Коммит создан

**Сообщение**: feat: добавить контент-слой
**Файлы** (N):
- src/lib/utils/parseContent.ts
- ...
**Хэш**: abc1234
```
