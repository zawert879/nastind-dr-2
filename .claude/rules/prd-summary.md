# PRD Summary (выжимка)

## Продукт

- **Название**: Модные Сучки
- **Цель**: Глянцевый mock-журнал для компании друзей — стильные статьи о каждом участнике, полностью статичный, без бэкенда
- **Аудитория**: 5–10 участников и их окружение; просмотр с мобильного

## Стиль

**Dark Editorial** — чёрный фон, жирная типографика (serif для заголовков, sans для текста),
акценты через градиенты hot pink → electric purple → gold, glassmorphism-карточки, Y2K-luxe.

## Ключевые фичи (MVP)

1. **Главная страница** — обложка выпуска (героиня номера, hero-баннер) + сетка карточек статей
2. **Теги/рубрики** — фильтрация карточек на главной (Стиль, Карьера, Путешествия и др.)
3. **Страница статьи** — обложка, MD-контент (фото, таблицы), секция "Похожие статьи" (рандомные)
4. **Статичные комментарии** — читаемые, не интерактивные; данные из `comment-{slug}.md`
5. **Тёмная тема** — единственная тема, градиентные акценты по всему сайту
6. **Контент из MD-файлов** — статьи и комментарии в репо, собираются при деплое

## Контент-структура

```
src/content/
├── articles/
│   ├── {slug}.md         ← статья (frontmatter: title, subject, tags, cover, featured)
│   └── ...
└── comments/
    ├── comment-{slug}.md ← статичные комментарии (frontmatter: author, date, avatar)
    └── ...
```

### Frontmatter статьи

```yaml
---
title: "Заголовок статьи"
subject: "Имя участника"
tags: ["Стиль", "Карьера"]
cover: "/images/covers/{slug}.jpg"
featured: true   # обложка выпуска на главной (только одна)
date: "2026-04-01"
---
```

### Frontmatter комментария

```yaml
---
article: "{slug}"
comments:
  - author: "Имя"
    avatar: "/images/avatars/name.jpg"
    date: "2026-03-28"
    text: "Текст комментария"
---
```

## Ключевые экраны

- `/` — Главная: hero-обложка выпуска + сетка карточек с фильтром по тегам
- `/articles/[slug]` — Статья: обложка, MD-контент, комментарии, похожие статьи

## Ограничения

- Только фронтенд, без SSR, без бэкенда
- Без авторизации
- Без внешних запросов в рантайме (всё собирается статично)
- Комментарии — только чтение, не интерактивные
- Поиск и шаринг не нужны

## Приоритеты

1. Структура контента (MD + frontmatter) и парсинг при билде
2. Главная страница с hero и сеткой карточек
3. Страница статьи с MD-рендером и комментариями
4. Фильтр по тегам
5. Секция "Похожие статьи"
6. Визуальная полировка (градиенты, типографика, анимации)
